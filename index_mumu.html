MUMU Challenge 2026

Mobile Unified Multimodal Understanding Challenge

Efficient Unified Multimodal Architectures for Mobile-Oriented Visual Understanding

Challenge Overview

The Mobile Unified Multimodal Understanding (MUMU) Challenge 2026 is a benchmark competition designed to advance research on efficient unified multimodal architectures for mobile-oriented visual understanding.


Unlike existing task-specific benchmarks, this challenge requires participants to design a single unified model that simultaneously performs semantic tagging, open-vocabulary object detection, and image caption generation under constrained computational budgets.


The challenge emphasizes algorithmic efficiency, architectural unification, and deployability on resource-constrained environments, while evaluation is conducted on PC-side compute statistics rather than requiring on-device execution.

Objectives

  • Promote unified vision-language modeling
  • Encourage efficient multi-task representation learning
  • Benchmark trade-offs between performance and complexity
  • Advance mobile-friendly multimodal perception systems

Competition Tasks

Task A: Multi-Concept Image Tagging

Participants must predict semantic concept labels from image quality attributes, scene categories, and event categories.

  • Image quality attributes: blur, noise, exposure, compression artifacts
  • Scene categories: indoor/outdoor, office, kitchen, street, stadium, etc.
  • Event categories: concert, wedding, sports event, protest, festival, etc.

Task B: Open-Vocabulary Object Detection

Participants must detect objects from a provided open vocabulary. Models must generalize to unseen categories.

  • Bounding box coordinates
  • Confidence score
  • Text label

Task C: Efficient Image Captioning

Participants must generate concise natural language descriptions for each input image.

  • Maximum 30 generated tokens
  • English only
  • Single-sentence output

Data splits include a training set of approximately 1.2M images, a 20,000-image validation set, a 20,000-image public test set, and a 30,000-image private final test set. Novel detection categories are hidden in private evaluation.

Rules & Evaluation

Model Constraints

  • Single unified model only
  • No task-specific independent models
  • Parameter Count ≤ 350M
  • GFLOPs ≤ 25 @ 224×224 input
  • Peak Inference Memory ≤ 8GB (PC-side)
  • Single forward pass per image

Evaluation Metrics

  • Task A: Macro-F1
  • Task B: Novel-aware mAP@[0.5:0.95]
  • Task C: 0.4 CIDEr + 0.3 SPICE + 0.3 CLIPScore
  • Final Score = (0.3A + 0.4B + 0.3C) × E
  • Efficiency factor: E = 1 / (1 + alpha * GFLOPs + beta * Params)

Ranking Rules

  • Primary ranking: Final Score
  • Tie-break 1: Lower GFLOPs
  • Tie-break 2: Lower parameter count
  • Tie-break 3: Higher detection performance

Submission Format

Participants must submit a ZIP archive containing model weights, configuration, inference entrypoint, dependencies, and documentation. 

submission.zip
model/
  weights.pt
  config.yaml
inference.py
requirements.txt
README.md

A unified predictor API is required. The predictor should initialize from a weight path and return tags, detections, and a caption for each input image.
